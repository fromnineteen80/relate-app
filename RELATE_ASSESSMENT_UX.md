# RELATE Assessment User Experience

## Overview

RELATE is a relationship compatibility assessment delivered as a Claude artifact. Users complete ~375 questions across 4 sections (~1 hour 45 minutes total). Progress saves automatically, with a backup string system for reliability.

**Time Breakdown:**
| Section | Questions | Time |
|---------|-----------|------|
| Demographics | ~15-20 | ~4 min |
| Session 1: Who You Want | 134 | ~38 min |
| Session 2: Who You Are | 137 | ~39 min |
| Session 3: Connect & Conflict | 88 | ~25 min |
| **Total** | **~375** | **~106 min** |

*Time estimates based on Enneagram benchmark: 144 questions in 40 minutes = ~17 seconds per question*

**Core UX Principles:**
- Never lose user progress
- Respect user time (clear session boundaries)
- Minimal friction to resume
- No account required beyond Claude

## Checkpoint String System

### User-Facing Explanation

**In the checkpoint screen, explain:**

> **ðŸ’¾ Backup Code**
> 
> Your progress saves automatically to your Claude account. This code is a backup in case something goes wrong.
> 
> **When you might need it:**
> - Progress doesn't load when you return
> - You're accessing Claude from a different device
> - You want to save your progress somewhere else
> 
> **How to use it:**
> 1. Copy the code and save it (notes app, email to yourself, etc.)
> 2. When you return, click "Enter code to resume" on the welcome screen
> 3. Paste your code and continue where you left off

### Code Display Best Practices

1. **Make it easy to copy:**
   - Single tap/click copies entire code
   - "Copied!" confirmation
   - Code displayed in monospace font

2. **Don't overwhelm:**
   - Truncate display with "..." if very long
   - Full code still copies correctly
   - Show character count: "347 characters"

3. **Emphasize it's optional:**
   - "Your progress is saved automatically"
   - "This is a backup, just in case"

### Storage Keys

```javascript
// Primary storage (automatic)
relate_gender          // 'M' or 'F'
relate_status          // 'single' or 'relationship'

relate_m3_responses    // JSON: { questionId: response, ... }
relate_m4_responses    // JSON: { questionId: response, ... }
```

| Event | Action |

| Question answered | Save current module responses |
| Session completed | Update session number, generate checkpoint code |
| "Take a Break" clicked | Ensure all data saved, show code prominently |
| Assessment complete | Generate and save results code |

## Mobile Considerations

1. **Touch targets:** Minimum 44x44px for all interactive elements
2. **Progress bar:** Sticky at top, doesn't scroll away
3. **Question text:** Large, readable (18px minimum)
4. **Response options:** Full-width buttons, generous padding
5. **Checkpoint code:** 
   - "Tap to copy" clearly indicated
   - Visible confirmation when copied
   - Option to share via native share sheet
6. **Keyboard:** Avoid unnecessary keyboard triggers (no text input except code entry)

---

## Accessibility

1. **Screen readers:**
   - Progress announced: "Question 47 of 134"
   - Response options clearly labeled
   - State changes announced

2. **Keyboard navigation:**
   - Tab through options
   - Enter/Space to select
   - Arrow keys to navigate options

3. **Color:**
   - Don't rely on color alone for state
   - Sufficient contrast (WCAG AA minimum)

4. **Motion:**
   - Respect prefers-reduced-motion
   - Auto-advance can be disabled

## Dating Overlay Concept

The core questions in `relate_questions.js` work for both single and partnered users. The **dating overlay** applies to single users and affects:

1. **Question Framing:** Some questions may be presented differently
   - Single: "In your ideal partner..."
   - Partnered: "In your current relationship..."

2. **Demographics Interpretation:** 
   - Single: Used for market analysis and pool calculations
   - Partnered: Used for context but not pool analysis

3. **Report Generation:**
   - Single: Includes market position, search channels, dating coaching
   - Partnered: Replaces with relationship-focused insights

4. **Results Code:**
   - Single: "Share this with future partners"
   - Partnered: "Share this with your partner to unlock compatibility report"

**The underlying assessment data is identical.** Only the presentation layer changes.

## Technical Notes for JSX Implementation

### Storage API Pattern

```javascript
// Save
const saveProgress = async (key, data) => {
  try {
    await window.storage.set(key, JSON.stringify(data));
    return true;
  } catch (e) {
    console.warn('Storage unavailable:', e);
    return false;
  }
};

// Load
const loadProgress = async (key) => {
  try {
    const result = await window.storage.get(key);
    return result ? JSON.parse(result.value) : null;
  } catch (e) {
    console.warn('Storage read failed:', e);
    return null;
  }
};
```

### Checkpoint String Pattern

```javascript
// At checkpoint
const generateCheckpoint = (gender, session, allResponses) => {
  const code = encodeCheckpointString(gender, session, allResponses);
  saveProgress('relate_checkpoint', code);
  return code;
};

// On resume
const restoreFromCode = (code, expectedGender) => {
  const result = decodeCheckpointString(code, expectedGender);
  if (result.valid) {
    // Restore responses to state
    // Navigate to appropriate session
  }
  return result;
};
```

## Summary: User Instructions (Plain Language)

### Taking the Assessment

1. Click "Begin Assessment"
2. Select your gender and relationship status
3. If in a relationship, optionally enter your partner's results code
4. Complete demographics questions (~4 minutes)
5. Answer assessment questions honestlyâ€”there are no wrong answers
6. At each checkpoint, you can continue or take a break
7. Your progress saves automatically

### Resuming Later

**If your progress loads automatically:**
- Click "Continue" and pick up where you left off

**If your progress doesn't load:**
1. Click "Enter code to resume"
2. Paste the backup code you saved
3. Click "Resume"

### Saving Your Backup Code

At each checkpoint, you'll see a backup code. Save it somewhere:
- Screenshot it
- Copy to a notes app  
- Email it to yourself

This code lets you resume even if something goes wrong.

### Sharing Results

**If you're single:**
- Your results code can be shared with future partners
- They can enter it when taking RELATE to see compatibility

**If you're in a relationship:**
- Share your results code with your partner
- When they complete RELATE, they enter your code
- Both of you get a full compatibility report
